# Markdown & Version Control 핵심 정리

## 1. Markdown 개념

### Markdown이란?
- README는 보통 **Markdown(.md)** 형식으로 작성된다.
- Markdown은 **텍스트를 구조화해 가독성을 높이는 문법**이다.
- 별도의 파일 형식이 아니라 **확장자만 다른 일반 텍스트 파일**이다.

### Markdown의 목적
- 읽기 쉽다.
- 쓰기 쉽다.

### Markdown이 사용되는 곳
- Wikipedia
- Tistory
- Notion
- GitHub

---

## 2. GitHub에서의 README

- GitHub는 GFM (GitHub Flavored Markdown)을 사용한다.
- `README.md` 파일은 **저장소 메인 화면에 자동으로 표시**된다.
- 프로젝트를 설명하는 **표준 문서 역할**을 한다.

---

## 3. 버전 관리 개념

### 버전이란?
버전은 **의미 있는 수정 기록**을 의미한다.

#### 버전으로 보지 않는 경우
- 단순 오타 수정
- 글자 하나 변경

#### 버전으로 보는 경우
- 기능 추가
- 기능 수정
- 성능 개선
