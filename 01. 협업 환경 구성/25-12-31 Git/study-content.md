# Git 핵심 요약

## 1. Git이란?

- **분산 버전 관리 시스템(DVCS)**
- 하나의 **프로그램**으로, **로컬 컴퓨터에 설치**되어 동작
- 프로젝트(폴더 단위)의 **변경 이력(버전)**을 관리한다

### Git의 역할
- 파일을 복사하지 않고 버전 관리 가능
- 수정 / 삭제 / 추가 내역 기록
- 특정 시점의 상태로 되돌리기 가능
- 협업을 위한 기반 제공

---

## 2. Git 사용 흐름

1. 로컬 폴더(프로젝트)를 Git으로 관리
2. 변경 사항을 Git에 기록
3. 기록된 변경 사항을 필요에 따라 공유 또는 백업

---

## 3. CLI와 GUI

### CLI (Command Line Interface)
- **명령어를 직접 입력**해 컴퓨터에 명령
- 셸(Shell) 환경에서 동작

#### 대표적인 셸
- 명령 프롬프트 (cmd)
- PowerShell
- Git Bash
- Terminal (macOS / Linux)

### GUI (Graphical User Interface)
- 마우스와 버튼을 사용하는 그래픽 환경
- 직관적이지만 세부 제어에는 한계가 있음

---

## 4. Git 기본 개념

- Git은 **처음부터 모든 파일을 자동으로 추적하지 않는다**
- 추적하려는 파일을 **명시적으로 지정**해야 한다
- Git은 **프로젝트(폴더) 단위**로 버전을 관리한다

---

## 기본 터미널 / Git 명령어 정리

---

### 1. 터미널 기본 명령어

- `ls`  
  → 현재 위치한 경로의 폴더 안에 있는 **파일 / 폴더 목록 조회**

- `mkdir 폴더명`  
  → 현재 위치한 경로에 **입력한 이름의 폴더 생성**

- `cd 폴더명`  
  → 입력한 폴더 경로로 이동

- `cd ..`  
  → 현재 폴더의 **상위 폴더로 이동**

- `pwd`  
  → 현재 위치한 **절대 경로 확인**

---

### 2. Git 기본 명령어

- `git init`  
  → 현재 폴더를 **새로운 Git 저장소로 초기화**  
 
- `git status`  
  → 현재 Git 상태 확인  
  → Git 저장소인지 확인  

- `git add 파일명`  
  → 입력한 파일을 **Git이 추적(tracking)하도록 요청**  

- `git add .`  
  → 현재 폴더의 **모든 변경 파일을 한 번에 추가**

- `git commit -m "메시지"`  
  → 지금까지 `add`된 변경 사항을  
    **하나의 버전(스냅샷)으로 생성**

- `git log`  
  → 지금까지 저장된 **버전(커밋) 기록 확인**
