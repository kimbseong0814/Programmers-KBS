# 웹 개발 직무 이해

## 우리가 개발하는 웹 서비스의 범위

웹 서비스를 개발할 때 **인터넷 통신망(광케이블, 라우터, ISP 등)** 자체를 구축하거나 코딩하지는 않는다.  
하지만 **인터넷 위에서 동작하는 통신 규칙과 구조**는 명확히 이해하고 사용한다.

즉,

- ❌ 물리적인 인터넷 망 구축 → 하지 않음  
- ⭕ 인터넷 위에서 동작하는 **HTTP 통신 구조** → 직접 사용  

웹 개발은 **웹 브라우저(Client)** 와 **웹 서버(Server)** 사이의 동작을 구현하는 일이다.

---

## 프론트엔드 (Frontend · Client-side)

프론트엔드는 웹 서비스에서 **사용자 측(Client-side)** 에서 동작하는 영역이다.

### 역할

- 화면(UI) 구성
- 사용자 입력 처리
- 서버와 통신하여 데이터 요청 및 응답 처리
- 결과를 화면에 출력

### 예시

- 글자 입력
- 버튼 클릭
- 로그인 화면 표시
- 서버 응답에 따른 화면 변경

### 사용 기술

- HTML
- CSS
- JavaScript

---

## 백엔드 (Backend · Server-side)

백엔드는 웹 서비스에서 **서버 측(Server-side)** 에서 동작하는 영역이다.

### 역할

- 프론트엔드에서 전달된 요청 처리
- 비즈니스 로직 수행
- 데이터베이스 조회 및 저장
- 인증 / 인가 처리
- 처리 결과를 프론트엔드에 전달

### 사용 기술 (예시)

- 서버 언어: Java, Python, Node.js 등
- 데이터베이스: MySQL, PostgreSQL 등
- 서버 프레임워크

---

## 1. 클라이언트(Client)와 서버(Server)

### 클라이언트(Client)

- **서비스를 요청하는 컴퓨터**
- 사용자가 직접 사용하는 쪽

### 예시

- 웹 브라우저 (Chrome, Safari, Edge)
- 스마트폰 앱
- PC 프로그램

---

### 서버(Server)

- **서비스를 제공하는 컴퓨터**
- 클라이언트의 요청을 처리하고 결과를 응답

### 예시

- 로그인 처리
- 게시글 저장
- 사용자 정보 조회

---

## 2. 클라이언트와 서버 사이의 약속: 프로토콜(Protocol)

### 프로토콜이란?

- 서로 정보를 주고받기 위해 **반드시 지켜야 하는 규칙**
- 데이터의 형식, 순서, 의미를 미리 정해둔 약속

---

## 3. 우리가 웹을 사용하는 방식 (HTTP)

### HTTP란?

- 웹에서 사용하는 대표적인 통신 프로토콜
- 웹 브라우저(클라이언트) ↔ 웹 서버 간 데이터 통신 규칙

### 특징

- 요청(Request) / 응답(Response) 구조
- **무상태(stateless) 프로토콜**

---

### 웹 동작 흐름

1. 사용자가 웹 주소 입력
2. 브라우저가 서버에 HTTP 요청
3. 서버가 요청 처리
4. 서버가 HTML / CSS / 데이터 응답
5. 브라우저가 화면에 표시

---

## 4. 웹 브라우저와 웹 서버의 역할

### 웹 브라우저

- 서버에서 받은 데이터를 화면으로 보여주는 프로그램
- HTML 해석
- CSS 적용
- JavaScript 실행

---

### 웹 서버

- 클라이언트 요청 수신
- 필요한 데이터 처리
- 결과를 다시 클라이언트에 전달

---

## 5. 프론트엔드와 백엔드의 협업

### 로그인 예시 흐름

#### 1️⃣ 프론트엔드

- 아이디 / 비밀번호 입력
- 로그인 버튼 클릭
- 서버로 로그인 요청 전송

#### 2️⃣ 백엔드

- 아이디 존재 여부 확인
- 비밀번호 비교
- 인증 결과 판단

#### 3️⃣ 결과 반환

- 성공 → 환영 페이지 데이터 전달
- 실패 → 오류 메시지 전달
