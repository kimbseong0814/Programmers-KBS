# 강의 내용 정리 (26.01.29)

TCP / IP & ASC / DSC 정리
1. TCP / IP란?

TCP/IP는 네트워크에서 데이터가

누가 누구에게 보내는지

데이터가 순서대로 도착했는지

중간에 유실되지는 않았는지

를 관리하기 위한 통신 규약(프로토콜) 묶음이다.

2. IP의 역할 — “누가 누구에게”

IP(Internet Protocol) 는
데이터를 어떤 컴퓨터에서 → 어떤 컴퓨터로 보낼지 결정한다.

모든 장치는 고유한 IP 주소를 가짐

발신자 IP와 수신자 IP를 기준으로 데이터 전달

데이터는 패킷 단위로 전송됨

👉 IP는 목적지 식별을 담당한다.

3. TCP의 역할 — “순서·유실·도착 여부 확인”

TCP(Transmission Control Protocol) 는
전송된 데이터가 정확하게 도착했는지를 책임진다.

데이터 전송 순서 보장

데이터 유실 여부 확인

누락된 데이터가 있으면 재전송 요청

수신자가 데이터를 받았다는 응답(ACK) 확인

통신 전후로 연결 설정/해제 수행

👉 TCP는
순서, 유실 여부, 정확한 도착을 모두 관리한다.

4. TCP / IP 동작 흐름 요약

IP로 목적지(누구에게 보낼지) 결정

TCP로 연결 설정

데이터를 여러 패킷으로 분할하여 전송

TCP가 패킷 순서와 유실 여부 확인

누락된 데이터는 재전송

모든 데이터가 도착하면 연결 종료

5. ASC / DSC란?

ASC / DSC는 데이터 조회 시 사용하는 정렬 방식이다.

ASC (Ascending)

오름차순 정렬

작은 값 → 큰 값

과거 → 현재

ORDER BY id ASC

DSC (Descending)

내림차순 정렬

큰 값 → 작은 값

최신 → 과거

ORDER BY created_at DESC
