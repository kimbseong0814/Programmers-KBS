# TCP / IP & ASC / DSC 정리 (26.02.03)

## 1. TCP / IP란?

**TCP/IP**는 네트워크에서 데이터가

- 누가 누구에게 보내는지

- 데이터가 순서대로 도착했는지

- 중간에 유실되지는 않았는지

-> 관리하기 위한 **통신 규약(프로토콜) 묶음**이다.

---
## 2. IP의 역할 

**IP(Internet Protocol) 는**
데이터를 **어떤 컴퓨터에서 → 어떤 컴퓨터로** 보낼지 결정한다.

- 모든 장치는 고유한 **IP 주소**를 가짐

- 발신자 IP와 수신자 IP를 기준으로 데이터 전달

- 데이터는 패킷 단위로 전송됨

-> IP는 **목적지 식별**을 담당한다.

---
## 3. TCP의 역할 

**TCP(Transmission Control Protocol) 는**
전송된 데이터가 **정확하게 도착했는지**를 책임진다.

- 데이터 **전송 순서 보장**

- 데이터 **유실 여부 확인**

- 누락된 데이터가 있으면 **재전송 요청**

- 수신자가 데이터를 받았다는 **응답(ACK)** 확인

- 통신 전후로 **연결 설정/해제** 수행

-> TCP는

**순서, 유실 여부, 정확한 도착**을 모두 관리한다.

---
## 4. TCP / IP 동작 흐름 요약

1. IP로 **목적지(누구에게 보낼지)** 결정

2. TCP로 연결 설정

3. 데이터를 여러 패킷으로 분할하여 전송

4. TCP가 패킷 **순서와 유실 여부 확인**

5. 누락된 데이터는 재전송

6. 모든 데이터가 도착하면 연결 종료

---
## 5. ASC / DSC란?

**ASC / DSC는 데이터 조회 시 사용하는 정렬 방식이다.**

### ASC (Ascending)

- 오름차순 정렬

- 작은 값 → 큰 값

- 과거 → 현재

```sql
ORDER BY id ASC
```

### DSC (Descending)

- 내림차순 정렬

- 큰 값 → 작은 값

- 최신 → 과거

```sql
ORDER BY created_at DESC
```
