# 강의 내용(26.01.30)

## Database & RDBMS 기본 개념 정리
### 1️. 데이터베이스(Database)란?

**데이터베이스(Database)** 는
- 데이터를 **통합하여 효율적으로 관리**하기 위한 **데이터 집합체**이다.

- 단순히 데이터를 저장하는 공간이 아니라,

  **검색·연산·관리**를 전제로 설계된 구조이다.

---
## 2️. 데이터베이스의 장점

- 데이터를 **구조화**하여 관리함으로써

   - 데이터 **중복을 방지**할 수 있다.

   - 데이터 **일관성**을 유지할 수 있다.

- 대량의 데이터를

  - **빠르고 효율적으로 연산**할 수 있다.

  - 조건 검색, 정렬, 집계 등이 가능하다.

---
## 3️. DBMS(Database Management System)란?

DBMS는
  - 데이터베이스를 생성, 관리, 운영하기 위한 소프트웨어이다.
  - 우리는 DBMS를 통해서만 데이터베이스를 사용한다.

### 대표적인 DBMS 종류

- Oracle

- MySQL

- MariaDB

---
## 4️. SQL이란?

SQL(Structured Query Language) 은
  - 데이터베이스에 **연산을 요청하기 위해 사용하는 언어**이다.

데이터를

- 생성(Create)

- 조회(Read)

- 수정(Update)

- 삭제(Delete)
  할 수 있다.

### 대표적인 SQL 명령어

```
기능	      명령어

데이터 삽입	INSERT

데이터 조회	SELECT

데이터 수정	UPDATE

데이터 삭제	DELETE
```
---
## 5️. RDBMS(Relational DBMS)

- **RDBMS**는 데이터를 **테이블(Table)** 형태로 관리하는 DBMS이다.

- 테이블 간의 관계를 통해 데이터를 연결한다.

---
## 6️. Key(키)의 개념

**Primary Key (PK, 기본 키)**

 - 테이블의 각 행(row) 을 유일하게 구별할 수 있는 값
 - 중복 ❌, NULL ❌

**Foreign Key (FK, 외래 키)**

 - 다른 테이블의 **데이터를 참조하기 위해 사용하는 키**
 - 일반적으로
     - **B 테이블의 PK**

     - → **A 테이블의 FK**로 사용한다.

---
## 7️. 테이블 간 관계 (연관관계)

RDBMS에서는 테이블 간에 다음과 같은 관계가 존재한다.

- 1 : 1

- 1 : N

- M : N

---
## 8️. 예시: 게시글(Post)과 사용자(User)
### 관계 분석

8-1 사용자 1명 → 게시글 여러 개
- 1 : N

8-2 게시글 1개 → 작성자 1명
- 1 : 1

**테이블 분리**

- User 테이블

- Post 테이블
post 테이블에 **User의 PK를 FK**로 저장한다.

---
## 9️. 관계 방향의 중요성

- 게시글 테이블 → 사용자 정보 조회 ⭕
   - (FK를 통해 사용자 식별 가능)

- 사용자 테이블 → 게시글 전체 조회 ❌
    - (별도의 조회 쿼리가 필요)

**연관관계는 FK가 있는 테이블 기준으로 따라간다.**

---
## 정리

- RDBMS는

  - 테이블을 분리하여 중복을 제거하고

  - 테이블 간의 관계(FK) 를 통해 데이터를 연결한다.

- 이를 통해

  - 데이터 무결성 유지

  - 효율적인 데이터 관리

  - 확장성 있는 구조를 만들 수 있다.
