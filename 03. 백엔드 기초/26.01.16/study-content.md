# 오늘의 강의 정리 (26.01.16)

## 모듈(Module)이란?

**기능별로 코드를 분리해 놓은 파일 또는 코드 묶음**이다.

- 하나의 파일 = 하나의 모듈
- 필요한 기능만 불러와서 사용 가능
- 코드 재사용성과 유지보수성 향상

### Node.js에서의 모듈

- `require()` : 모듈 불러오기
- `module.exports` : 외부로 내보내기

 Node.js는 **모듈 기반 구조**로 동작한다.

---

## Method

메서드: **객체에 소속된 함수**를 의미한다.

### 특징

- 특정 객체의 기능을 수행
- 객체의 데이터와 함께 동작

### 예시 개념

- `console.log()`
    
    → `console` 객체에 포함된 `log` 메서드
    

함수는 독립적, 메서드는 **객체에 종속**

---

## Node.js의 특징

### ① 비동기(Non-blocking) 처리

- 오래 걸리는 작업을 기다리지 않고 다음 코드 실행
- `setTimeout`, 파일 I/O, 네트워크 요청 등에 사용

### ② 싱글 스레드 기반

- 하나의 메인 스레드
- 이벤트 루프를 통해 여러 작업 처리

### ③ 서버 사이드 JavaScript

- 브라우저가 아닌 서버에서 JS 실행
- 웹 서버, API 서버 구현 가능

### ④ 모듈 기반 구조

- 필요한 기능만 불러서 사용
- 경량화된 서버 구현 가능

## 라이브러리와 프레임워크

### 라이브러리

- **필요한 기능만 가져다 씀**
- 제어 흐름은 개발자가 관리

예: 날짜 처리, HTTP 요청, 유틸 함수 모음

---

### 프레임워크

- **정해진 구조 안에서 개발**
- 제어 흐름을 프레임워크가 관리

예: 서버 구조, 라우팅, 전체 프로젝트 뼈대 제공

---

### 핵심 차이

- 라이브러리: *내가 필요할 때 호출*
- 프레임워크: *프레임워크가 나를 호출*

---

## 콜백 함수

콜백 함수: **다른 함수의 인자로 전달되어, 특정 시점에 실행되는 함수**다.

### 사용 이유

- 비동기 작업 처리
- 작업 완료 후 실행할 동작 정의

### 예시 개념

- `setTimeout(함수, 시간)`
    
    → 시간이 지난 후 함수 실행
    

Node.js 비동기 처리의 **기본 구조**

---

## NPM이란?
**Node.js에서 사용하는 패키지(라이브러리) 관리 도구**다.

- Node.js 설치 시 **자동 포함**
- 외부 모듈을 **설치·관리·공유**하는 역할
- 세계 최대 규모의 JavaScript 패키지 저장소를 사용
