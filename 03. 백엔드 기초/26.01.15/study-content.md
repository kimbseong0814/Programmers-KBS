# 백엔드 API와 HTTP 요청 기본 개념 정리

## 1. API란 무엇인가

**API (Application Programming Interface)**  
서로 다른 프로그램이 정해진 방식으로 데이터를 요청·응답하기 위한 규칙이다.

### 예시
* 지하철 도착 정보 앱
* 카카오맵 / 네이버 지도

실제 데이터는 **기관(예: 서울교통공사 DB)**에 존재하며,  
앱은 API를 통해 필요한 데이터만 요청하여 사용한다.

---

## 2. Interface의 의미

**Interface**란  
서로 다른 두 대상 사이에서 중간 매개체 역할을 한다.

* 사용자 ↔ 프로그램
* 프로그램 ↔ 서버
* 서버 ↔ 데이터베이스

직접 접근하지 않고, **정해진 통로(규칙)**를 통해서만 소통한다.

---

## 3. GUI와 CLI

### GUI (Graphical User Interface)
* 컴퓨터(프로그램)한테 명령을 내릴 때, 그래픽을 사용해서 명령을 내리는 방식

### CLI / HTTP 요청 관점
* 서버에게 데이터 처리 요청을 보내는 방식
* 웹에서는 **URL + HTTP Method**로 요청 전달
* 단순 위치(URL)뿐 아니라 행동 요청까지 포함

---

## 4. URL과 HTTP Method 개념

* URL은 자원의 위치를 의미
* 행동은 HTTP Method로 표현

### 예시 (상품 API)

```html
  기능          HTTP   Method   URL
* 상품          등록    POST    /products
  
* 전체 상품     조회     GET    /products
  
* 전체 상품     삭제    DELETE  /products

```
URL에 동작(등록, 조회, 삭제)을 넣지 않는다  
동작은 HTTP Method로 표현한다

---

## 5. URL 설계 규칙 

### 기본 규칙
* 대문자 사용 ❌ → 소문자 사용 ⭕
* 언더바(_) ❌ → 하이픈(-) ⭕
* 마지막 `/` 포함 ❌
* 파일 확장자(.html, .jsp) 포함 ❌
* 행위를 포함하지 않습니다 = 목적을 포함하지 않습니다
* 복수형 사용 ⭕
